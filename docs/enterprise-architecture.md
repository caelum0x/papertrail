# PaperTrail — Enterprise Architecture Blueprint

_Generated by a 7-agent planning workflow (domain/product · API+data · agentic/AI · enterprise infra → synthesize → adversarial review → finalize). The build executes this wave by wave._

**Targets:** 133 pages · 221 API routes · 25 modules.

## Vision

PaperTrail becomes an enterprise evidence-verification and research platform for life-sciences: a multi-tenant system where pharma medical-affairs, HEOR, regulatory/medical-writing, systematic-review teams, CROs, academic labs, and pharmacovigilance groups verify scientific and efficacy claims against primary sources, ingest and mine their own document corpora, and run auditable agentic research workflows. The existing deterministic core is preserved and elevated as the trust differentiator: every claim is checked against ClinicalTrials.gov structured registered results and raw event counts with code-side recomputation of ARR/NNT/RR+CI (lib/biostats.ts), multi-source cross-verification, and code-enforced exact-span grounding (lib/grounding.ts) — no AI assertion survives unless it maps to a verbatim source substring; below the similarity threshold the system returns no_support_found rather than a forced match. CRITICAL BASELINE CORRECTION (verified against the tree): the current app is NOT greenfield-auth. It already ships session auth (pt_session httpOnly JWT via jose/HS256, lib/auth/session.ts, middleware.ts guarding /console/*, /api/auth/{login,logout,register,session}), multi-tenancy (orgs, memberships[owner/admin/editor/viewer], invitations, audit_log, api_keys — every migration 0001-0009 already carries org_id), coarse RBAC (lib/authz/rbac.ts role-rank matrix + requireRole), and 39 API routes + 31 pages. Wave 1 is therefore an UPGRADE, not greenfield: session->SSO, coarse 4-role->permission matrix, add Postgres RLS (absent today), upgrade the flat audit_log to a hash-chained WORM chain, replace the in-memory single-instance rate limiter with distributed Redis, and split the sources table into a global read-only registry cache vs tenant-owned uploads. On top of that hard core sit enterprise foundations (org tenancy with RLS, RBAC permission matrix, SSO/SCIM, immutable 21 CFR Part 11-grade audit + e-signature bound to the audit chain), domain workspaces (systematic review/PRISMA, evidence library, references, publication planning, pharmacovigilance), a composable agent workflow engine bridging to Anthropic's Claude Science workbench (beta connector, no hard upstream dependency), and a commercial surface (billing/usage metering with quota-authorization BEFORE Claude spend, public API, developer portal, integrations). The result is the verification and provenance layer of record for regulated evidence work — Covidence/DistillerSR/Veeva-class capability with a deterministic adjudication engine at its center.

## Personas

- Pharma medical-affairs / med-comms lead — plans publications and MLR-reviews scientific statements, each backed by a verified citation
- HEOR analyst — runs systematic literature reviews and builds evidence tables for value dossiers
- Regulatory / medical-writing specialist — needs 21 CFR Part 11 audit trail, e-signatures, and provenance for submissions
- Systematic-review methodologist — runs PRISMA-aligned dual screening, RoB 2, and GRADE synthesis
- CRO integration engineer — embeds PaperTrail verification into external pipelines via the public API and webhooks
- Academic translational researcher — verifies manuscript/grant claims against primary sources and manages references (core hackathon audience per CLAUDE.md)
- Pharmacovigilance officer — runs scheduled safety-literature monitors and triages adverse-event signals
- Organization admin — manages SSO/SCIM, members, roles, security policy, and billing for their tenant
- Platform operator (Anthropic/Gladstone staff) — cross-tenant support, audited impersonation, feature flags, and system health

## Build waves

- **Wave 1 — Foundation UPGRADE (not greenfield — auth/orgs/RBAC/audit_log/api_keys/39 routes/31 pages already exist). Ship the four load-bearing corrections: (1) Postgres RLS + mandatory tenant DB client — currently absent, must precede route sprawl; (2) split registry_sources from tenant sources; (3) upgrade flat audit_log to hash-chained WORM — e-sig and every mutation depend on it; (4) distributed Redis rate limiting + durable job queue (batch verify off the request path) + quota-authorization-before-Claude in the verify path. Also: session->SSO, coarse 4-role -> permission matrix. After this wave the existing PaperTrail flow works org-scoped, RLS-isolated, authenticated, audited, quota-guarded, with batch verification off the request path.**: Auth & Identity, Organizations, Tenancy & RBAC, Projects & Workspaces, Claims & Verification (Deterministic Core), Audit Trail, Compliance & Provenance, Background Jobs, Queues & Schedules, Account, Profile & Onboarding
- **Wave 2 — Evidence platform + enterprise trust layer: cache/retrieval/search (registry vs org split, consolidated search endpoints), document ingestion (R2 + Docling + Claude PDF), the reusable evidence/reference library, org-facing security/isolation controls (RLS itself already shipped Wave 1), and platform admin/observability.**: Sources, Retrieval & Evidence Search, Document Ingestion & Extraction, Evidence Library & References, Security, Data Isolation & Residency, Platform Admin & Observability
- **Wave 3 — Research workflows + collaboration + commercialization: PRISMA systematic review, the agentic workflow engine, human review/approval with e-sign (binds to the Wave-1 audit chain), reporting/analytics, the public API/developer portal (rate limiting already shipped Wave 1), and — SPLIT — Billing & Usage Metering separate from Notifications.**: Systematic Review & Screening, Agentic Workflow Engine & Agent Library, Review, Collaboration & Approvals, Reporting, Exports & Analytics, API Platform & Developer Portal, Billing & Usage Metering, Notifications
- **Wave 4 — Advanced agentic + domain depth + growth surface, with explicit deferability. Claude Science bridge (highest-risk, Anthropic-hosted beta, NO hard upstream dependency) and Publication Planning (sales-driven, weakest fit for translational-lab audience) are both explicitly deferrable if the timeline slips. Plus MCP/tool governance, agent evals (sole owner of the eval entity cluster), pharmacovigilance, integrations marketplace, and the marketing/docs/trust center (a 4-endpoint CMS, not a full module).**: Claude Science Integration, MCP & Tool Registry, Agent Evaluation & Quality, Pharmacovigilance & Literature Monitoring, Publication Planning & Medical Writing, Integrations, Marketing, Docs & Trust Center

## Tech additions (OSS/libraries)

- Auth: WorkOS AuthKit for enterprise SSO (SAML/OIDC), SCIM directory sync, and MFA, layered ON TOP of the existing pt_session cookie + middleware.ts guard (session auth already exists — this is an upgrade, not a replacement)
- Tenant isolation (WAVE 1, CRITICAL): Postgres RLS — currently absent — via ENABLE+FORCE ROW LEVEL SECURITY on every tenant table + a mandatory lib/db/tenantClient.ts that sets SET LOCAL app.current_org per transaction and is the ONLY DB-handle path. registry_sources (global cache) split from tenant-owned sources so no permissive NULL-org carve-out is ever needed
- Job queue / durable workflows: Trigger.dev v4 (Apache-2.0, self-hostable, dedicated long-running compute, human-in-the-loop waitpoint tokens with callback URLs for review/e-sign gates). CAVEAT: warm-starts/checkpoints/auto-scaling are Cloud-only — self-hosting forgoes checkpoint/resume unless paying for Cloud
- Redis (Upstash) — WAVE 1 — for distributed rate limiting (replacing the in-memory single-instance lib/rateLimit.ts which is non-functional multi-region), token buckets per org/API key, and Trigger.dev backing store
- Object storage: Cloudflare R2 (S3-compatible) for uploaded PDF blobs, per-org KMS envelope encryption + signed expiring URLs
- PDF parsing: Docling (IBM, open-source scientific-paper layout/table/formula extraction) for deterministic parse, paired with native Claude PDF (Files API, page-level citations, 32MB/100pp limits) for cited semantic extraction
- XML/registry parsing: fast-xml-parser for PubMed E-utilities XML and ClinicalTrials.gov v2 responses
- Billing: Stripe Billing for subscriptions/seats/invoices/payments + an internal usage_events table as metering source of truth (quota authorization BEFORE expensive Claude chains — the authorize-before-Claude control ships in Wave 1). Orb/Metronome deferred until event volume warrants SQL-defined rating
- Agent orchestration: Claude Agent SDK for the workflow engine (coordinator + specialist sub-agents, tool allow-lists, MCP) and PaperTrail-as-MCP-server exposing verify/statAudit/search to Claude Science (beta connector, no hard dependency)
- Transactional email: Resend (or Postmark) for invites, assignments, job-completion, quota/PV alerts, digests — driven off the Wave-1 job queue (Notifications module, split from Billing)
- Observability: OpenTelemetry (vendor-neutral traces/metrics/logs) to OpenObserve (self-hosted, OTel-native) + Sentry for errors; privacy-preserving structured logging preserved (never log claim text or keys, extending lib/logger.ts)
- Secrets management: Doppler (or AWS Secrets Manager) with required-secret validation at startup, extending the existing .env.example discipline
- Validation: Zod remains the mandatory LLM trust boundary (lib/schemas.ts) — no raw JSON.parse of any LLM output; extended to every new agent. Advisory LLM checks (citation-qa, reviewer-agent, llm-judge) NEVER gate — deterministic lib/grounding.ts is the pass/fail gate
- Database: Postgres + pgvector on Neon retained (vector(1024) matches existing lib/embeddings.ts Voyage contract); add pg RLS policies + BM25/keyword index (pg_trgm or ParadeDB pg_search) for hybrid retrieval with reciprocal-rank fusion
- Citations: citation-js / CSL for AMA/Vancouver/APA/custom-style bibliography formatting in the References module
